# Changelog

## v1.5.13 (2026-04-26)

### Refactor: Move Core Types to /types Directory

- **Moved** all core type definitions from `src/core/types.ts`, `src/core/hooks.ts`, `src/core/plugin-meta.ts`, and `src/core/plugin.ts` to new files in `src/types/`:
  - `src/types/pipeline.ts` (core types: PipelineFile, PipelineResult, Storage, PipelineContext, Validator, Processor, PipelineConfig, OwnedProcessor, OwnedValidator)
  - `src/types/hooks.ts` (PipelineHooks)
  - `src/types/plugin-meta.ts` (PluginMeta, PluginTraceEvent, PipelineMeta)
  - `src/types/plugin.ts` (PipelinePlugin, PipelinePluginSetup, isPipelinePlugin)
- **Deleted** the original type files from `src/core/`.
- **Updated** all imports in core, storage, processor, and validator modules to reference the new `src/types/` files.

### API/Behavior Changes

- `executePipeline` now merges `ctx.metadata` and `result.metadata` in the returned result for more complete metadata propagation.
- Improved error message for invalid plugin objects in `createPipeline`.

### Internal/Codebase

- All modules now import types from `src/types/` instead of `src/core/types.ts` and related files.
- Documentation and module references updated to reflect new type locations.

### No Breaking Changes to Public API

- All public APIs remain unchanged, but type imports for consumers should now use the new paths if importing directly from source.

---

**Migration Notes:**
- If you import types directly from `src/core/types.ts`, update your imports to use `src/types/pipeline.ts`, `src/types/hooks.ts`, `src/types/plugin-meta.ts`, or `src/types/plugin.ts` as appropriate.
- No changes required for users importing from the main package entry point.
