# Changelog

All notable changes to **media-pipline** will be documented in this file.

The format is based on Keep a Changelog, and this project follows semantic versioning.

---

## [1.4.4] - 2026-04-24

### Added
- Initial **plugin system (`.use()`)**
- `PipelineBuilder` introduced as internal composition layer
- Ability for plugins to:
  - Add validators
  - Add processors
  - Extend hooks
- Hook system introduced with lifecycle support:
  - `onStart`
  - `afterValidate`
  - `afterProcess`
  - `onFinish`
  - `onError`

- Hook merging system with chaining support:
  - Multiple plugins can attach to the same hook without overwriting each other
- Processor pipeline chaining support:
  - Multiple processors executed sequentially
  - Each processor receives and returns `ctx`
- Validator pipeline execution before processing
- Storage abstraction layer integrated into pipeline execution

### Changed
- Refactored `createPipeline` to use `PipelineBuilder` as single source of truth
- Execution flow now uses builder state instead of static config
- Improved pipeline execution flow: 
```validate â†’ process â†’ store â†’ return result```


### Fixed
- Fixed issue where plugin validators and processors were not executed
- Fixed hook execution order inconsistencies
- Fixed hook merging behavior to support multiple plugins correctly
- Fixed processor execution not reflecting in final output (metadata now preserved)
- Fixed TypeScript typing conflicts in hook merging logic using controlled casting

### Improved
- Stronger pipeline lifecycle control
- Better separation between:
- execution context (`ctx`)
- storage output (file result)
- metadata (processing result)
- Improved debugging visibility during pipeline execution
