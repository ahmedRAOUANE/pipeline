# Changelog

## v1.6.1 (2026-04-27)

### Features

- **Enhanced local storage functionality**: Added support for ensuring storage directory is writable before saving files
- **Improved plugin type definitions**: Added `displayName` property to `PipelinePluginFunction` for better plugin identification
- **Plugin validation enhancements**: Added plugin utilities and improved error handling for invalid plugins
- **Function plugins support**: Enhanced plugin system to support function-based plugins

### Refactor

- **Project structure reorganization**: Enhanced plugin type validation and reorganized project structure
- **Import cleanup**: Cleaned up unused imports and improved plugin name handling in `createPipeline`

### Fixes

- **Error handling in executePipeline**: Ensured `onError` hook execution is properly traced
- **Storage directory validation**: Added check to ensure storage directory is writable before saving files

### Documentation

- Added comprehensive documentation for pipeline modules and types
- Restored docs folder with full documentation structure

---

## v1.6.0 (2026-04-26)

### Features

- **Complete documentation overhaul**: Added comprehensive documentation for all aspects of the project
  - Architecture documentation (design patterns, system architecture)
  - API documentation (endpoints, request/response schemas)
  - Data flow documentation (overview, request lifecycle)
  - Module documentation (all core modules)
  - Dependencies documentation (npm packages, external services)
  - Development documentation (environment, setup)
  - Appendix (assumptions, glossary)

### Refactor

- **Type definitions centralization**: Moved all core type definitions to `src/types/` directory
  - `src/types/pipeline.ts` - Core pipeline types
  - `src/types/hooks.ts` - Pipeline hooks types
  - `src/types/plugin-meta.ts` - Plugin metadata types
  - `src/types/plugin.ts` - Plugin types

### Internal

- Updated all imports to use new type locations
- Improved error messages for plugin validation
- Enhanced metadata merging in pipeline execution

---

**Migration Notes (v1.5.x → v1.6.x):**

- If you import types directly from `src/core/types.ts`, update your imports to use:
  - `src/types/pipeline.ts` for core types
  - `src/types/hooks.ts` for hook types
  - `src/types/plugin-meta.ts` for plugin metadata types
  - `src/types/plugin.ts` for plugin types
- No changes required for users importing from the main package entry point
