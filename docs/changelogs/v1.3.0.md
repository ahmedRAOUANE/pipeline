# Changelog

All notable changes to the **Media Pipeline** project will be documented in this file.

The format is based on Keep a Changelog, and this project follows semantic versioning.

---

## [1.3.0] - 2026-04-23

### 🚀 Added

#### Core Pipeline System
- Introduced full **pipeline execution engine**
- Implemented structured flow:
  - Input → Validation → Processing → Storage → Output

#### Hook System (Lifecycle Events)
Added extensible lifecycle hooks system:

- `onStart(ctx)`
- `afterValidate(ctx)`
- `afterProcess(ctx)`
- `onError(error, ctx)`
- `onFinish(result, ctx)`

This enables:
- Observability
- Debugging
- Logging integration
- Plugin-ready architecture foundation

#### Error System
- Introduced structured error classes:
  - `PipelineError`
  - `ValidationError`
  - `ProcessorError`
  - `StorageError`

- Added:
  - Error `code` system
  - Error `details` metadata
  - Type-safe error handling via `instanceof`

#### Storage System
- Added `localStorage` adapter
- Supports saving files to local filesystem
- Designed for future storage adapters (S3, MinIO, etc.)

#### Validators
- Added built-in validators:
  - `maxSize(limit)`
  - `allowedMimeTypes(types[])`

#### Processors
- Added `identityProcessor` (baseline passthrough processor)
- Processor pipeline fully composable and sequential

---

### 🔧 Changed

- Updated package build system to support dual module output:
  - CommonJS (`.js`)
  - ES Modules (`.mjs`)
- Updated `tsup` configuration for multi-format builds
- Improved TypeScript declaration generation (`.d.ts` included in build output)
- Refined `exports` field in `package.json` for proper ESM/CJS resolution

---

### 📦 Build & Packaging

- Added `"files": ["dist"]` to ensure clean npm packaging
- Ensured correct export mapping:
  - `import` → ESM (`dist/index.mjs`)
  - `require` → CJS (`dist/index.js`)
  - `types` → TypeScript declarations

---

### 🐞 Fixed

- Fixed ESM/CJS interoperability issues
- Fixed `instanceof PipelineError` breaking due to module duplication
- Fixed missing TypeScript declaration issues in consumer projects
- Fixed runtime export inconsistencies between builds
- Fixed hook execution ordering issues during pipeline lifecycle

---

### 🧠 Internal Improvements

- Introduced clean separation of concerns:
  - Core pipeline engine
  - Validators
  - Processors
  - Storage adapters
  - Hooks system
  - Error system

- Improved execution stability and determinism
- Improved plugin-readiness of architecture

---

### ⚠️ Known Limitations

- No async queue system yet (all processing is synchronous in sequence)
- No plugin system (manual wiring only)
- No streaming support (buffer-based only)
- No retry mechanism for failed operations
